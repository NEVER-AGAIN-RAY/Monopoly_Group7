# Monopoly Deal Cards Game 需求文档

## 1. User Requirement (用户需求)
* **游戏模式**：系统应允许2-5名玩家进行游戏，玩家可自由选择与AI或真实玩家对战。
* **交互体验**：玩家应能通过图形用户界面（GUI）完成摸牌、出牌、支付租金及管理财产等交互操作。
* **信息透明**：玩家应能实时查看公共牌堆状态、自身资产及对手的公开资产信息，同时确保手牌隐私。
* **自动判定**：系统应自动执行复杂的收租计算、规则验证，并在达成胜利条件时立即通知玩家。
* **进度管理**：系统应允许玩家在不同模式下保存游戏进度，以便后续恢复。

---

## 2. System Requirement (系统需求)

### 2.1 Functional Requirement (功能需求)

#### 一、游戏基础启动与初始化
* **模式选择**：系统应当提供“人机对战”与“玩家对战”模式入口。在人机模式下，系统应当支持玩家设置“简单/普通/困难”三种不同的AI难度。
* **人数与顺序**：系统应当支持2-5人创建新游戏，并随机指定一名先手玩家，回合按顺时针轮转。
* **资源初始化**：游戏启动时，系统应当：
    1. 按照108张标准配置生成牌堆。
    2. 自动为每位玩家分发5张初始手牌。
    3. 向玩家展示抽牌堆剩余数量并初始化空的弃牌堆。
* **规则查看**：系统应当集成游戏规则面板，支持玩家在游戏内随时调出查看。

#### 二、回合逻辑与动作控制
* **摸牌机制**：玩家在自己的回合内，首先必须从公共抽牌堆摸取2张牌，若玩家摸牌前手牌为空，则系统应当允许摸取5张牌。摸牌完成后，系统自动更新玩家手牌数量，并实时刷新界面的手牌展示区域。
* **出牌功能**：玩家完成摸牌后，可执行出牌动作，可选择将手中的卡牌放置到自己的”财产区“”行动区“存入个人银行或直接弃牌，每回合最多可打出三张牌（所有出牌行为累计）。支持的操作包括：
    * **存入银行**：允许玩家将钱币卡、行动卡存入个人银行堆，且存入后的行动卡失去原有功能，仅作为钱币结算使用。
    * **部署房产**：允许玩家将手牌中的房产卡放置到自身财产区，用于构建同色房产集；系统自动对财产区卡牌按颜色归类展示。
    * **执行行动**：允许玩家将手牌中的行动卡打出至游戏中心行动区，触发卡牌对应的特殊效果（如收租、偷牌、免租等）；打出后系统自动执行效果并更新游戏状态。
* **回合结束与弃牌**：玩家完成摸牌、出牌所有操作后，进入回合结束弃牌阶段，若此时玩家手牌数量超过7张，系统应当强制玩家弃牌至抽牌堆，直至手牌剩余7张。弃牌完成后，系统自动结束当前玩家回合，切换至下一位玩家。

#### 三、游戏核心机制
* **收租逻辑**：系统需根据玩家财产区中同色房产集的完成度（基础房产 / 带房子 / 带酒店），按照游戏规则自动计算租金金额，并向指定目标玩家发起收租指令。
* **支付处理**：
    * 系统允许玩家通过个人银行堆的钱币卡、或将财产区的房产卡退回弃牌堆的方式支付租金欠款，禁止使用手牌直接支付。
    * 系统必须执行”找零不退“的规则：若玩家支付的卡牌总价值超过应付租金金额，超出部分不作找零，直接归收租方所有。
    * 若被收租玩家的个人银行堆无可用钱币卡、且财产区无可退回的房产卡，则该玩家无需支付本次租金，收租方本次收租操作失败。
* **万能牌处理**：房产万能牌可替代任意颜色的房产卡，用于补齐同色房产集；玩家可在自己的回合内，自由调整财产区中房产万能牌的位置，调整完成后，系统立即重新计算对应房产集的完成度，并更新租金计算标准。
* **卡牌出牌合法性验证**：
    * 系统对玩家的所有出牌行为进行规则验证，对非法操作（如无对应房产集时打出收租牌、超出每回合 3 张出牌限制、手牌为空时弃牌等）进行即时拦截，并弹出清晰的错误提示（如 “当前无可用房产集，无法打出收租牌”）。
    * 玩家选择行动卡目标时，系统验证目标的合法性（如偷牌需选择有手牌 / 银行卡的玩家、拆毁房产需选择有已部署房产的玩家），对无效目标进行拦截。
* **抽牌堆补堆机制**：当玩家摸牌时，若公共抽牌堆无可用卡牌，系统自动将弃牌堆所有卡牌随机洗牌后，转化为新的公共抽牌堆，保证游戏正常进行；补堆完成后向所有玩家展示抽牌堆新的剩余数量。

#### 四、游戏状态反馈与操作提示
* **实时状态反馈**：
    * 系统在游戏界面实时展示当前回合玩家、抽牌堆剩余数量、弃牌堆数量；
    * 每位玩家的界面区域，实时展示自身手牌数量、财产区卡牌（按颜色归类）、个人银行堆卡牌数量及总面值；
    * 其他玩家的公开信息（财产区卡牌、银行堆总面值）正常展示，手牌为私密信息，仅展示数量不展示具体卡牌。
* **操作即时提示**：
    * 玩家执行摸牌、出牌、弃牌等操作后，系统通过文字提示 + 卡牌动画反馈操作结果（如“玩家 A 摸取2张牌，当前手牌数：6”）；
    * 当玩家有可执行操作（如摸牌、出牌）时，系统对对应操作按钮进行高亮提示，引导玩家操作。

#### 五、游戏胜利与结束判定
* **胜利判定**：系统实时检测所有玩家的财产区状态，当某一玩家集齐 3 套及以上完整的同色房产集（含使用万能牌补齐的房产集），立即判定该玩家为游戏胜利者，终止游戏进程并展示胜利界面。
* **强制结束判定**：若游戏中出现所有玩家主动选择退出、或单局游戏时长超过1小时，系统强制结束游戏，展示当前游戏进度（各玩家房产集完成情况、银行堆总面值）。

#### 六、游戏暂停与进度保存
* **人机对战模式**
    * **暂停**：玩家可随时暂停，冻结游戏界面，暂停后可直接继续 / 保存退出 / 放弃退出，无额外确认步骤。
    * **进度保存**：支持手动保存（任意时刻可操作）+自动保存（每 3 个完整回合自动覆盖最新进度）；启动游戏可直接加载本地保存进度，一键恢复所有游戏状态（玩家 / AI / 牌堆 / 回合）。
* **玩家对战模式（多人）**
    * **暂停**：仅当前回合玩家可发起暂停，需所有玩家确认同意才生效；暂停后仅支持“全员继续游戏”或“全员保存并退出”，无单人放弃选项。
    * **进度保存**：仅支持全员确认手动保存（当前回合玩家发起，所有人同意后存储），无自动保存；加载进度需参与保存的所有玩家确认，否则无法加载。
