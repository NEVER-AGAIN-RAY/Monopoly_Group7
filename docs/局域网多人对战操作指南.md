# 局域网多人对战操作指南（一台电脑当服务器）

本说明面向「宿舍 / 教室 / 小组演示」场景：**不需要云服务器**，用任意一台电脑当「房主主机」跑游戏服务，其他人用自己的电脑连进来一起玩。  
照着步骤做即可；若某一步和你的系统界面不一致，优先在系统设置里搜「防火墙」「网络」等关键词找同类选项。

---

## 你需要准备什么


| 项目  | 说明                                                   |
| --- | ---------------------------------------------------- |
| 网络  | 所有电脑连接 **同一个 Wi‑Fi**（或同一有线网）。手机热点也可以，但让所有玩家都连这个热点。   |
| 软件  | 每台电脑都能运行本项目的客户端（已安装 **JDK 17**、**Maven**，会执行文档里的命令）。 |
| 人数  | 游戏支持 2～5 名真人玩家（开局时人数要与实际一致）。                         |


---

## 第一步：选一台电脑当「主机」（跑服务器）

这台电脑会一直处于「开服」状态，建议选 **电量充足** 或 **插电** 的笔记本，关闭休眠以免断线。

### 1.1 打开终端，进入项目文件夹

在终端里进入你们克隆下来的项目根目录（里面有 `pom.xml` 的那个文件夹）。

### 1.2 启动服务器

执行：

```bash
mvn -q exec:java
```

看到类似 **「Monopoly WebSocket running at ws://localhost:8025/ws」** 的提示就表示服务已启动。  
**这个窗口不要关**，关掉就等于关服。

### 1.3 记下这台主机的「局域网 IP 地址」

其他玩家连接时需要填主机的 IP，**不是** `localhost`。

- **Windows**：设置 → 网络和 Internet → WLAN（或以太网）→ 点当前网络 → 查看「IPv4 地址」，常见形如 `192.168.x.x`。
- **macOS**：系统设置 → 网络 → 选当前 Wi‑Fi → 看「IP 地址」。
- **命令行（可选）**：Windows 用 `ipconfig`，macOS/Linux 用 `ifconfig` 或 `ip addr`，在结果里找 `192.168.` 或 `10.` 开头的那一段。

**假设主机 IP 是 `192.168.1.100`**，下面举例都用这个（你们换成自己的）。

---

## 第二步：主机上放行防火墙（很重要）

否则别的电脑明明在同一个 Wi‑Fi，也会 **连不上**。

- **Windows**：控制面板 → Windows 安全中心 → 防火墙 → 高级设置 → **入站规则** → 新建规则 → 端口 → TCP → 特定本地端口填 **8025** → 允许连接 → 名称可写「Monopoly 游戏」。
- **macOS**：系统设置 → 网络 → 防火墙 → 防火墙选项 → 若 Maven/Java 弹窗询问是否允许传入连接，选择 **允许**。（不同版本菜单位置可能略有不同。）

做完这一步，再让其他玩家连接。

---

## 第三步：其他玩家电脑上的操作（客户端）

每台「参与游戏的电脑」都要装同一套项目，并能跑图形界面客户端。

### 3.1 启动游戏界面

在项目根目录执行：

```bash
mvn javafx:run
```

### 3.2 填写连接地址

在游戏界面里找到 **WebSocket 地址 / ws 地址** 一栏：

- 不要再用 `ws://localhost:8025/ws`（那表示「只连自己这台电脑」）。
- 改成：`**ws://主机的IP:8025/ws`**  
例如：`ws://192.168.1.100:8025/ws`

然后点击 **连接**。若显示已连接，说明网络通了。

### 3.3 每位玩家设置不同的「玩家编号」

选择游戏模式为 **PVP（人人对战）** 后，界面里会有提示：大家要分别用 `**pvp-1`、`pvp-2`、`pvp-3`** ……

- 第 1 位玩家：把玩家 ID 设为 `pvp-1`，点 **认证**（或界面上的「认证并开局」之前先完成认证，见下一步）。
- 第 2 位玩家：`pvp-2`，认证。
- 依此类推。

**不要两个人都用同一个编号**，否则游戏里会乱套。

---

## 第四步：开局顺序（所有人要先连上、先认证）

**推荐流程：**

1. 主机先开服（见第一步），保持运行。
2. **所有玩家** 都先 **连接** 成功，并完成各自 `**pvp-x` 的认证**。
3. 约定好由 **其中一位**（一般是主机玩家）负责点 **开局 / START**：
  - 模式选 **PVP**。
  - **玩家数量** 填实际有几个人（例如 3 人就填 3）。
4. 只点 **一次** 开局即可，所有客户端都会收到游戏状态更新。

若有人还没连上或未认证就开局，可能出现有人进不了局、或操作对不上号的情况；出现问题时先 **全员重连 + 由房主重新开一局** 再试。

---

## 第五步：主机自己也想一起玩怎么办？

可以。同一台电脑上：

1. 一个终端窗口继续跑 `mvn -q exec:java`（服务器）。
2. **再开一个终端**（新开窗口），同样在项目里执行 `mvn javafx:run` 打开客户端。
3. 客户端地址填 `ws://localhost:8025/ws`，玩家 ID 填 `pvp-1`（或你们约定的编号之一），再和大家一起认证、开局。

---

## 常见问题排查


| 现象                            | 可能原因                                      | 可以尝试                          |
| ----------------------------- | ----------------------------------------- | ----------------------------- |
| 别人填了 `ws://主机IP:8025/ws` 仍连不上 | 防火墙没放行、或 IP 抄错、或不在同一 Wi‑Fi                | 核对 IP；检查防火墙；用手机热点时确保所有人都连该热点  |
| 连上了但操作不像自己                    | 玩家 ID 和别人重复了                              | 每人用不同的 `pvp-1`、`pvp-2`… 并重新认证 |
| 主机休眠/断网                       | 笔记本合盖或省电断网                                | 插电、关闭休眠、主机保持亮屏或调整电源选项         |
| 想换一局                          | 由一位玩家在规则允许范围内发起新局（或全员退出客户端后重连），按「第四步」重新开局 |                               |

## 单人模式（人机对战）快速开局

如果只是一个人想玩，不需要找队友，选人机模式即可——你 vs AI，全程在本机完成。

### 步骤

1. **启动服务器**

   在项目根目录打开终端，执行：

   ```bash
   mvn -q exec:java
   ```

   看到 `Monopoly WebSocket running at ws://localhost:8025/ws` 即可。

2. **启动客户端**

   再开一个终端窗口，执行：

   ```bash
   mvn javafx:run
   ```

3. **在界面上配置并开始**

   - **游戏模式**：选 **HVM**（Human vs Machine）。
   - **玩家人数**：保持默认 2 即可（你 + 1 个 AI）；也可以选 3～5，多出来的全是 AI。
   - **AI 难度**：EASY / NORMAL / HARD 三档，按喜好选。
   - **语言**：可选中文或 English。
   - 直接点 **「开始游戏」**，客户端会自动连接本机服务器、认证、开局，无需手动填地址或玩家 ID。

4. **开始游戏**

   进入游戏后按照界面引导操作：摸牌 → 出牌 → 结束回合。AI 会在你结束回合后自动行动。

### 注意事项

- 不需要改服务器地址，默认 `ws://localhost:8025/ws` 就是连本机。
- 不需要手动认证或填玩家 ID，点「开始游戏」一键搞定。
- 如果想重开一局，关闭客户端窗口后重新 `mvn javafx:run` 即可（服务器不用重启）。

---

## 和「人机对战」的差别（一句话）

- **人机**：一般是 **一个人** 开客户端，模式选 HVM，人类编号多为 `human-1`，电脑自动扮演 AI。  
- **人人**：**每个人** 一台客户端，模式选 PVP，每人用自己的 `pvp-x`，**没有 AI**，全靠真人操作。

---

## 附录：用命令行快速测服务器有没有通（可选）

会用的同学可以在任意电脑上装 `wscat`，测试与主机的端口是否通：

```bash
wscat -c ws://192.168.1.100:8025/ws
```

连接成功后发一行（可把内容复制粘贴）：

```json
{"type":"PING","payload":{}}
```

若有回复，说明网络与端口正常，再开游戏客户端即可。

*文档对应项目内 WebSocket 默认端口 **8025**、路径 `**/ws*`*；若你们修改了代码中的端口，请把本文所有 `8025` 改成实际端口。*
